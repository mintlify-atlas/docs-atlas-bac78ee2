## Backend structure

### Entry point

**`api.py`** - FastAPI application
- Initializes database on startup
- Configures CORS middleware
- Registers routers for `/api/summarize` and `/api/chats`
- Health check endpoint at `/health`

### Core services (`core/`)

Business logic separated from API layer:

```text
core/
в”њв”Ђв”Ђ config.py           # Settings and logging configuration
в”њв”Ђв”Ђ extractor.py        # Text extraction from PDF/EPUB/URLs
в”њв”Ђв”Ђ summarizer.py       # LLM-based summarization
в”њв”Ђв”Ђ pdf_export.py       # PDF generation for summaries
в”њв”Ђв”Ђ auth.py             # Better Auth integration helpers
в””в”Ђв”Ђ dependencies.py     # FastAPI dependency injection
```

**`config.py:12`** - Settings class
- Loads environment variables
- Defines configuration constants
- Provides cached settings instance
- Configures structured logging

**`extractor.py`** - Content extraction
- `extract_from_pdf()` - PyMuPDF text extraction
- `extract_from_scanned_pdf()` - OCR with Tesseract
- `extract_from_epub()` - EPUB content parsing
- `extract_url_content()` - Web page and file extraction
- `extract_book_content()` - Main entry point with fallback

**`summarizer.py:85`** - AI summarization
- `generate_summary()` - Sends text to OpenRouter API
- `parse_summary_json()` - Validates LLM response
- `SYSTEM_PROMPT` - Detailed instructions for structured output
- Returns metadata, summary, and chapter breakdowns

**`pdf_export.py`** - Export functionality
- Generates formatted PDF from summary JSON
- Preserves structure and formatting

**`auth.py`** - Authentication
- `get_current_user_id()` - Validates session and extracts user ID
- `get_optional_user_id()` - Non-required auth for public endpoints

SQLAlchemy models and repositories:

**Models** (`db/models/`)

- `Summary` - Stores book content and generated summaries
- `UserSummary` - Links users to their saved summaries  
- `Chat` - Chat sessions for books
- `Message` - Individual messages in chat history

**Repositories** (`db/repositories/`)

Data access layer implementing repository pattern:

- `SummaryRepository` - Create, retrieve, cache, user library
- `ChatRepository` - Create chats, add messages, update titles

### API routes (`routers/`)

FastAPI routers organized by domain:

```text
routers/
в”њв”Ђв”Ђ __init__.py         # Router exports
в”њв”Ђв”Ђ summarize.py        # Book upload and summary endpoints
в””в”Ђв”Ђ chat.py             # Chat creation and messaging
```

**`routers/summarize.py`**

- `POST /api/summarize` - Upload and summarize book
- `GET /api/summary/{id}` - Retrieve summary by ID
- `GET /api/summary/{id}/pdf` - Download summary as PDF
- `POST /api/summary/{id}/save` - Save to user library
- `DELETE /api/summary/{id}/save` - Remove from library
- `GET /api/library` - List user's saved summaries

**`routers/chat.py`**

- `POST /api/chats` - Create new chat for a book
- `GET /api/chats` - List user's chats
- `GET /api/chats/{id}` - Get chat with message history
- `POST /api/chats/{id}/messages` - Send message (streaming response)
- `DELETE /api/chats/{id}` - Delete chat

### RAG system (`rag/`)

Retrieval-Augmented Generation implementation:

```text
rag/
в”њв”Ђв”Ђ __init__.py         # Exports public functions
в”њв”Ђв”Ђ indexer.py          # Document chunking and indexing
в”њв”Ђв”Ђ vector_store.py     # ChromaDB collection management
в””в”Ђв”Ђ chat_engine.py      # Query engine with context retrieval
```

**`indexer.py:38`** - Book indexing

- `index_book()` - Chunks content, generates embeddings, stores vectors
- `is_indexed()` - Checks if book already indexed
- `delete_index()` - Removes book from vector store
- Uses LlamaIndex for chunking (1024 chars, 200 overlap)

**`vector_store.py`** - Vector database

- Manages ChromaDB collections (one per book)
- Creates, retrieves, and deletes collections
- Persistent storage in `data/chroma/`

**`chat_engine.py`** - Chat functionality

- Retrieves relevant chunks for user query
- Combines context with chat history
- Streams LLM responses
- Generates chat titles based on first message

## Data directories

Generated at runtime:

**`data/`** - Application data
- `app.db` - SQLite database (if using SQLite)
- `chroma/` - ChromaDB vector store

**`logs/`** - Application logs
- `app.log` - Rotating log files (5 MB max, 5 backups)

**`.env`** - Backend environment variables
- API keys (OpenRouter, OpenAI)

- Better Auth URL
- Log level

**`client/.env.local`** - Frontend environment variables

- Better Auth database URL

**`requirements.txt`** - Python dependencies
- FastAPI, SQLAlchemy, LlamaIndex
- PDF/EPUB libraries
- AI/ML libraries

**`client/package.json`** - Node.js dependencies
- Next.js, React, TypeScript
- Tailwind CSS, Better Auth

## Import conventions

### Backend

- Absolute imports from project root
- Example: `from core.config import get_settings`
- Models imported from `db.models`
- Repositories imported from `db.repositories`

### Frontend

- Relative imports for components
- Absolute imports from `lib/` directory
- Type imports from component files

## File naming conventions

### Backend
- Snake case: `vector_store.py`, `chat_engine.py`
- Single responsibility per file
- Clear, descriptive names

### Frontend
- PascalCase for components: `ChatInput.tsx`
- camelCase for utilities: `api.ts`, `auth-client.ts`
- Page files: `page.tsx` (Next.js convention)
