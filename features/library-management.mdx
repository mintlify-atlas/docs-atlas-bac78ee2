## Overview

Reading Assistant provides a personal library where you can save and manage your book summaries. Once saved, summaries remain accessible across sessions and devices.

## Saving summaries

To add a summary to your library:

```http
POST /summary/{summary_id}/save
```

**Response**:
```json
{
  "saved": true
}
```

This creates a `UserSummary` record linking your user ID to the summary. The same summary can be saved by multiple users without duplication.

<Info>
Saving a summary requires authentication. You must be logged in to use this feature.
</Info>

See `routers/summarize.py:139` for the save endpoint.

## Removing summaries

To remove a summary from your library:

```http
DELETE /summary/{summary_id}/save
```

**Response**:
```json
{
  "saved": false
}
```

<Note>
Removing a summary from your library does NOT delete the summary itself. It only removes the association with your account. The summary remains available for other users and can be re-saved later.
</Note>

See `routers/summarize.py:150` for the unsave endpoint.

## Listing your library

Retrieve all summaries you've saved:

```http
GET /library
```

**Response**:
```json
{
  "summaries": [
    {
      "id": "abc123",
      "metadata": {
        "title": "Deep Work",
        "author": "Cal Newport",
        "isAutoGenerated": false,
        "language": "en"
      },
      "summary": {
        "coreThesis": "...",
        "keyInsights": [...],
        "actionableTakeaways": [...],
        "readThisIf": "..."
      },
      "chapters": [...],
      "createdAt": "2024-01-15T10:30:00Z",
      "cached": true
    }
  ]
}
```

**Ordering**: Summaries are returned in reverse chronological order (most recently saved first).

**Timestamps**: The `createdAt` field shows when YOU saved the summary, not when it was originally created.

See `routers/summarize.py:161` for the library endpoint.

## Checking save status

When fetching a summary by ID, the response includes whether it's in your library:

```http
GET /summary/{summary_id}
```

**Response**:
```json
{
  "id": "abc123",
  "isSaved": true,
  "metadata": {...},
  "summary": {...}
}
```

The `isSaved` field is:
- `true` if the summary is in your library
- `false` if not saved
- `false` if you're not authenticated

This allows frontend applications to show "Save" or "Saved" buttons dynamically.

See `routers/summarize.py:95` for the get summary endpoint.

## Data model

The library system uses two database models:

### Summary model

Stores the actual book content and generated summary:

```python
class Summary(Base):
    __tablename__ = "summaries"
    
    id: str                 # UUID primary key
    content_hash: str       # SHA-256 hash for deduplication
    content: str            # Full book text
    summary: str            # JSON string with summary data
    created_at: datetime    # When summary was generated
```

**Relationships**:
- `user_summaries`: Links to users who saved this summary
- `chats`: Links to chat sessions for this book

See `db/models/summary.py:16` for the model definition.

### UserSummary model

Links users to their saved summaries:

```python
class UserSummary(Base):
    __tablename__ = "user_summaries"
    
    id: str                 # UUID primary key
    user_id: str            # User who saved the summary
    summary_id: str         # Foreign key to summaries table
    created_at: datetime    # When user saved the summary
```

**Indexes**: Both `user_id` and `summary_id` are indexed for fast queries.

**Relationship**: `summary` relationship loads the full Summary object.

See `db/models/summary.py:38` for the model definition.

## Repository methods

The `SummaryRepository` class provides methods for library management:

<Accordion title="save_for_user">
```python
def save_for_user(self, user_id: str, summary_id: str) -> UserSummary:
    """Save a summary to a user's collection."""
    user_summary = UserSummary(user_id=user_id, summary_id=summary_id)
    self.db.add(user_summary)
    self.db.commit()
    return user_summary
```

Creates a new `UserSummary` record linking the user to the summary. If already saved, this will create a duplicate (constraint violation should be handled by unique index).
</Accordion>

<Accordion title="unsave_for_user">
```python
def unsave_for_user(self, user_id: str, summary_id: str) -> None:
    """Remove a summary from a user's collection."""
    self.db.query(UserSummary).filter(
        UserSummary.user_id == user_id,
        UserSummary.summary_id == summary_id,
    ).delete()
    self.db.commit()
```

Deletes the `UserSummary` record. If not found, the delete operation does nothing (no error).
</Accordion>

<Accordion title="get_user_summaries">
```python
def get_user_summaries(self, user_id: str) -> list[dict]:
    """Get all summaries for a user."""
    user_summaries = (
        self.db.query(UserSummary)
        .filter(UserSummary.user_id == user_id)
        .order_by(UserSummary.created_at.desc())
        .all()
    )
    
    summaries = []
    for us in user_summaries:
        summary = self.get_by_id(us.summary_id)
        if summary:
            summary_dict = summary.get_summary_dict()
            summary_dict["createdAt"] = us.created_at
            summaries.append(summary_dict)
    
    return summaries
```

Joins `UserSummary` with `Summary` to return complete summary data with user-specific timestamps.
</Accordion>

<Accordion title="is_saved_by_user">
```python
def is_saved_by_user(self, summary_id: str, user_id: str | None = None) -> bool:
    """Check if a summary is saved by a user."""
    if not user_id:
        return False
    
    return (
        self.db.query(UserSummary)
        .filter(
            UserSummary.user_id == user_id,
            UserSummary.summary_id == summary_id,
        )
        .first()
        is not None
    )
```

Returns `True` if the user has saved the summary, `False` otherwise. Returns `False` if `user_id` is `None` (unauthenticated requests).
</Accordion>

See `db/repositories/summary.py` for all repository methods.

## Content deduplication

The library system uses content-based deduplication to avoid storing the same book multiple times:

**Hash computation**: SHA-256 hash of the book content

```python
def compute_hash(content: str) -> str:
    """Compute SHA-256 hash of content."""
    return hashlib.sha256(content.encode("utf-8")).hexdigest()
```

**Caching flow**:
1. User uploads a book
2. System computes content hash
3. Checks if a summary with this hash exists
4. If yes, returns cached summary immediately
5. If no, generates new summary and stores it

**Benefits**:
- Same book uploaded by different users = single summary stored
- Re-uploading the same book = instant result from cache
- Reduces LLM API costs and generation time

See `db/repositories/__init__.py:8` and `routers/summarize.py:72` for the caching implementation.

## Authentication

All library endpoints require authentication:

**Dependency**: `get_current_user_id` extracts the user ID from the request

**Header**: `Authorization: Bearer <token>`

**Response** (if not authenticated): HTTP 401 Unauthorized

```json
{
  "detail": "Not authenticated"
}
```

## Usage patterns

### Save after summarization

```bash
# 1. Summarize a book
response = requests.post(
    "https://api.readingassistant.com/summarize",
    files={"file": open("book.pdf", "rb")},
    headers={"Authorization": f"Bearer {token}"}
)
summary_id = response.json()["id"]

# 2. Save to library
requests.post(
    f"https://api.readingassistant.com/summary/{summary_id}/save",
    headers={"Authorization": f"Bearer {token}"}
)
```

### Check if saved before displaying

```bash
# Get summary with save status
response = requests.get(
    f"https://api.readingassistant.com/summary/{summary_id}",
    headers={"Authorization": f"Bearer {token}"}
)

if response.json()["isSaved"]:
    show_unsave_button()
else:
    show_save_button()
```

### Display library grid

```bash
# Fetch all saved summaries
response = requests.get(
    "https://api.readingassistant.com/library",
    headers={"Authorization": f"Bearer {token}"}
)

for summary in response.json()["summaries"]:
    display_book_card(
        title=summary["metadata"]["title"],
        author=summary["metadata"]["author"],
        saved_at=summary["createdAt"]
    )
```

## Related features

### Chat integration

When you create a chat for a book, the chat is linked to the summary:

```python
class Chat(Base):
    summary_id: str  # Foreign key to summaries table
    summary = relationship("Summary", back_populates="chats")
```

This means:
- You can chat with any book in your library
- Chats are associated with summaries, not user-specific copies
- Multiple users can chat with the same book (separate chat sessions)

### PDF export

Saved summaries can be exported as PDFs at any time using `/summary/{summary_id}/pdf`. The export doesn't require the summary to be in your library.
