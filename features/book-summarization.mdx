## Overview

Reading Assistant uses advanced language models to analyze your books and generate comprehensive summaries that help you understand the core concepts quickly. The system detects the book's language automatically and generates summaries in the same language.

## How it works

When you upload a book or provide text, the summarization engine:

1. **Extracts the content** - Reads text from PDF, EPUB, or plain text sources
2. **Analyzes the structure** - Identifies chapters, sections, and key themes
3. **Generates insights** - Uses AI to extract the core thesis, key insights, and actionable takeaways
4. **Returns structured data** - Provides a JSON response with metadata, summary, and chapter breakdowns

<Note>
The system can handle books up to 400,000 characters. Longer texts are automatically truncated while preserving the most important content.
</Note>

## Summary structure

Every generated summary follows a consistent structure designed to maximize learning:

```json
{
  "metadata": {
    "title": "Book Title",
    "author": "Author Name or null",
    "isAutoGenerated": false,
    "language": "en"
  },
  "summary": {
    "coreThesis": "One sentence capturing the central argument",
    "keyInsights": [
      "5-7 most important ideas, one sentence each"
    ],
    "actionableTakeaways": [
      "3-5 specific actions you can take"
    ],
    "readThisIf": "Who should read this and why"
  },
  "chapters": [
    {
      "number": 1,
      "title": "Chapter Title",
      "summary": "Detailed chapter breakdown"
    }
  ]
}
```

### Metadata

<Accordion title="Title and author detection">
The system attempts to extract the book title and author from the text itself (title pages, headers, "by [name]" patterns). If not found, it generates a descriptive title and sets the author to `null`. The `isAutoGenerated` field indicates whether the title was created by the AI.
</Accordion>

<Accordion title="Language detection">
The `language` field contains the ISO 639-1 language code (e.g., "en" for English, "am" for Amharic). All summary content is generated in this detected language to ensure accessibility.
</Accordion>

### Core thesis

A single sentence (maximum 30 words) that captures the book's central argument or main message. This gives you an instant understanding of what the book is about.

### Key insights

5-7 of the most important ideas from the book, each expressed in one sentence. These represent the core concepts you need to understand.

### Actionable takeaways

3-5 specific actions you can implement immediately. Each takeaway starts with a verb and focuses on practical application of the book's concepts.

### Read this if

1-2 sentences describing who would benefit from reading this book and why. Helps you determine if the book is relevant to your goals.

## Chapter summaries

Each chapter is broken down into four sections:

**Core Idea**: 1-2 sentences on the main argument of the chapter

**Key Points**: 2-3 bullet points highlighting the reasoning and evidence

**Example**: One concrete example or case study from the chapter

**Actionable Takeaway**: One specific thing you can apply immediately

<Info>
If the book has no clear chapter structure, the system creates 3-7 thematic sections with generated titles to organize the content logically.
</Info>

## Multilingual support

The summarization system supports books in any language:

- **Automatic detection** - Identifies the book's language from the content
- **Native summaries** - Generates all summary text in the detected language
- **Consistent structure** - JSON keys remain in English for API compatibility
- **Mixed content** - Handles books with multiple scripts (e.g., English with Amharic examples)

<Note>
For best results with non-Latin scripts (Arabic, Hebrew, Ethiopic, etc.), use PDF files with embedded fonts or EPUB format.
</Note>

## Smart caching

Summaries are cached based on content hash to improve performance and reduce costs:

- **Instant results** - Previously summarized books return immediately
- **Content-based** - Same content always returns the same summary
- **No duplicates** - Uploading the same book twice uses the cached version

You can check if a summary was cached by looking at the `cached` field in the response:

```json
{
  "id": "abc123",
  "cached": true,
  "metadata": { ... },
  "summary": { ... }
}
```

## Implementation details

The summarization engine is implemented in `core/summarizer.py:85` using the following approach:

**Model**: Configurable via `LLM_MODEL` setting (uses OpenRouter API)

**Temperature**: 0.3 (lower values produce more consistent, focused summaries)

**Max tokens**: 5,000 (sufficient for comprehensive book summaries)

**Timeout**: 120 seconds (handles large books gracefully)

The system uses a carefully crafted prompt (see `core/summarizer.py:19`) that instructs the AI to:
- Detect and match the input language
- Extract or generate metadata intelligently
- Create structured, actionable summaries
- Format chapter summaries consistently
- Skip non-content sections (prefaces, indexes, etc.)
