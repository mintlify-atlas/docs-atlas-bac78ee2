# Installation

This guide covers everything you need to install and configure Reading Assistant for local development or production deployment.

## Prerequisites

Ensure you have the following installed before proceeding:

  <Card title="Node.js 24+" icon="node-js">

## Backend setup

The backend is built with Python and FastAPI, handling all book processing, summarization, and API endpoints.

### Install dependencies

<Steps>
  <Step title="Create virtual environment">
    Set up an isolated Python environment:

    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    ```
  </Step>

  <Step title="Install packages with uv">
    Use the `uv` package manager for fast installation:

    ```bash
    uv pip install -r requirements.txt
    ```

    This installs all required dependencies:

    <CodeGroup>
    ```txt Core framework
    fastapi==0.128.0
    uvicorn==0.40.0
    python-multipart==0.0.21
    sqlalchemy==2.0.45
    python-dotenv==1.2.1
    httpx==0.28.1
    ```

    ```txt Document processing
    pymupdf==1.26.7
    ebooklib==0.20
    beautifulsoup4==4.14.3
    pdf2image==1.17.0
    pytesseract==0.3.13
    ```

    ```txt AI & RAG
    llama-index==0.14.12
    chromadb==1.4.1
    llama-index-vector-stores-chroma==0.5.5
    curl-cffi==0.14.0
    ```

    ```txt PDF export
    fpdf2==2.8.5
    reportlab==4.4.9
    ```
    </CodeGroup>
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root:

    ```bash
    cp .env.example .env
    ```

    Edit the `.env` file with your configuration:

    ```bash
    # API Keys (required)
    OPENROUTER_API_KEY=your_openrouter_api_key
    OPENAI_API_KEY=your_openai_api_key

    # LLM Configuration
    LLM_MODEL=google/gemini-3-flash-preview

    # Database
    DATABASE_URL=sqlite:///data/app.db

    # Auth
    BETTER_AUTH_URL=http://localhost:3000

    # Logging
    LOG_LEVEL=INFO
    ```

    <Warning>
      **Never commit your `.env` file to version control.** Keep your API keys secure.
    </Warning>
  </Step>
</Steps>

### Environment variables reference

| Variable | Description | Default |
|----------|-------------|----------|
| `OPENROUTER_API_KEY` | API key for OpenRouter (required for summarization) | None |
| `OPENAI_API_KEY` | API key for OpenAI embeddings (required for RAG) | None |
| `LLM_MODEL` | The LLM model to use via OpenRouter | `google/gemini-3-flash-preview` |
| `DATABASE_URL` | Database connection string | `sqlite:///data/app.db` |
| `BETTER_AUTH_URL` | Frontend URL for auth callbacks | `http://localhost:3000` |
| `LOG_LEVEL` | Logging verbosity (DEBUG, INFO, WARNING, ERROR) | `INFO` |
| `MAX_FILE_SIZE_MB` | Maximum upload file size in MB | `50` |
| `MIN_TEXT_LENGTH` | Minimum text length for summarization | `100` |
| `MAX_TEXT_LENGTH` | Maximum text length (will be truncated) | `500000` |
| `CORS_ORIGINS` | Allowed CORS origins (comma-separated) | `http://localhost:3000` |

### Database initialization

The database is automatically initialized when you start the backend server for the first time:

```python api.py:15-19
@asynccontextmanager
async def lifespan(app: FastAPI):
    """Initialize database on startup."""
    init_db()
    logger.info("Database initialized")
    yield
```

The SQLite database file will be created at `data/app.db` (or your configured path). For production, consider using PostgreSQL:

```bash
DATABASE_URL=postgresql://user:password@localhost:5432/reading_assistant
```

## Frontend setup

The frontend is built with Next.js 16 and React 19, providing a modern UI for book summarization.

<Steps>
  <Step title="Navigate to client directory">
    ```bash
    cd client
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```

    Key dependencies installed:

    <CodeGroup>
    ```json Framework
    {
      "next": "16.1.1",
      "react": "19.2.3",
      "react-dom": "19.2.3",
      "typescript": "^5"
    }
    ```

    ```json UI & Styling
    {
      "tailwindcss": "^4.1.18",
      "@tailwindcss/postcss": "^4.1.18",
      "@tailwindcss/typography": "^0.5.19",
      "lucide-react": "^0.562.0",
      "next-themes": "^0.4.6",
      "react-markdown": "^10.1.0"
    }
    ```

    ```json Auth & Database
    {
      "better-auth": "^1.4.17",
      "drizzle-orm": "^0.45.1",
      "drizzle-kit": "^0.31.8",
      "pg": "^8.17.2"
    }
    ```
    </CodeGroup>
  </Step>

  <Step title="Configure frontend environment">
    Create `client/.env.local`:

    ```bash
    # Backend API URL
    NEXT_PUBLIC_API_URL=http://localhost:8000

    # Database for Better Auth (PostgreSQL required)
    DATABASE_URL=postgresql://user:password@host:5432/dbname

    # Auth configuration
    BETTER_AUTH_SECRET=your-secret-key-here
    BETTER_AUTH_URL=http://localhost:3000
    ```

    <Note>
      Generate a secure `BETTER_AUTH_SECRET` using: `openssl rand -base64 32`
    </Note>
  </Step>

  <Step title="Setup PostgreSQL for auth">
    Better Auth requires a PostgreSQL database. You can use a managed service like [Neon](https://neon.tech/), [Supabase](https://supabase.com/), or run PostgreSQL locally.

    **Using Neon (recommended for development):**
    1. Create a free account at [neon.tech](https://neon.tech/)
    2. Create a new project
    3. Copy the connection string
    4. Add it to `DATABASE_URL` in `client/.env.local`

    **Using local PostgreSQL:**
    ```bash
    # Create database
    createdb reading_assistant_auth

    # Set connection string
    DATABASE_URL=postgresql://localhost:5432/reading_assistant_auth
    ```
  </Step>

  <Step title="Run database migrations">
    Push the auth schema to your PostgreSQL database:

    ```bash
    npx drizzle-kit push
    ```

    This creates the necessary tables for user authentication and sessions.
  </Step>
</Steps>

## Running locally

With both backend and frontend configured, you're ready to start the application.

### Start the backend

In your project root (with the virtual environment activated):

```bash
uvicorn api:app --reload --host 0.0.0.0 --port 8000
```

You should see output like:

```bash
INFO:     Database initialized
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
```

The API will be available at `http://localhost:8000`. Check the health endpoint:

```bash
curl http://localhost:8000/health
```

Expected response:
```json
{"status": "ok"}
```

### Start the frontend

In a new terminal, navigate to the client directory:

```bash
cd client
npm run dev
```

The frontend will start at `http://localhost:3000`.

<Note>
  The development server supports hot reloading. Changes to your code will automatically refresh the application.
</Note>

## Troubleshooting

### Backend issues

**Import errors:**
```bash
# Ensure virtual environment is activated
source venv/bin/activate

# Reinstall dependencies
uv pip install -r requirements.txt
```

**Database errors:**
```bash
# Remove and recreate database
rm -rf data/app.db
# Restart the server to reinitialize
```

**API key errors:**
```bash
# Verify .env file exists and contains valid keys
cat .env | grep API_KEY
```

### Frontend issues

**Build errors:**
```bash
# Clear node modules and reinstall
rm -rf node_modules package-lock.json
npm install
```

**Auth database errors:**
```bash
# Verify DATABASE_URL is correct
echo $DATABASE_URL

# Rerun migrations
npx drizzle-kit push
```

**CORS errors:**
Ensure your backend `.env` includes the frontend URL:
```bash
CORS_ORIGINS=http://localhost:3000
```
