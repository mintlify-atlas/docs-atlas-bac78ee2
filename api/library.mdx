## Save summary

<Card title="POST /api/summary/{summary_id}/save" icon="bookmark">
  Save a summary to your library
</Card>

Add a summary to the authenticated user's library for easy access later.

### Authentication

Required.

### Path parameters

<ParamField path="summary_id" type="string" required>
  The summary identifier to save
</ParamField>

### Response

<ResponseField name="saved" type="boolean">
  Always returns `true` on success
</ResponseField>

### Example

```bash
curl -X POST https://your-domain.com/api/summary/sum_abc123/save \
  -H "Authorization: Bearer <session-token>"
```

```json
{
  "saved": true
}
```

### Errors

<ResponseField name="401 Unauthorized">
  ```json
  {
    "detail": "Session token not found"
  }
  ```
</ResponseField>

<ResponseField name="404 Not Found">
  The summary doesn't exist (returns from repository layer)
</ResponseField>

## Unsave summary

<Card title="DELETE /api/summary/{summary_id}/save" icon="bookmark-slash">
  Remove a summary from your library
</Card>

Remove a previously saved summary from the user's library.

### Authentication

Required.

### Path parameters

<ParamField path="summary_id" type="string" required>
  The summary identifier to unsave
</ParamField>

### Response

<ResponseField name="saved" type="boolean">
  Always returns `false` on success
</ResponseField>

### Example

```bash
curl -X DELETE https://your-domain.com/api/summary/sum_abc123/save \
  -H "Authorization: Bearer <session-token>"
```

```json
{
  "saved": false
}
```

### Errors

<ResponseField name="401 Unauthorized">
  ```json
  {
    "detail": "Session token not found"
  }
  ```
</ResponseField>

## List library

<Card title="GET /api/library" icon="books">
  Get all summaries saved by the current user
</Card>

Retrieve all summaries that the authenticated user has saved to their library.

### Authentication

Required.

### Response

<ResponseField name="summaries" type="array">
  Array of summary objects. Each summary includes the full summary structure (metadata, summary, chapters) plus:
  
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Summary identifier
    </ResponseField>
    
    <ResponseField name="metadata" type="object">
      Book metadata (title, author, language, etc.)
    </ResponseField>
    
    <ResponseField name="summary" type="object">
      High-level summary (coreThesis, keyInsights, actionableTakeaways, readThisIf)
    </ResponseField>
    
    <ResponseField name="chapters" type="array">
      Array of chapter summaries
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      When the summary was originally created
    </ResponseField>
    
    <ResponseField name="saved_at" type="string">
      When the user saved this summary
    </ResponseField>
  </Expandable>
</ResponseField>

curl https://your-domain.com/api/library \

## Library behavior

### Save status

When retrieving a summary via `GET /api/summary/{id}`, the response includes an `isSaved` field indicating whether the current user has saved it to their library.

### Persistence

Saved summaries remain in the library until explicitly unsaved. Deleting the original summary does not automatically remove it from user libraries.

### Privacy

Users can only see and manage their own library. Library contents are private to each user.
