## Create summary

<Card title="POST /api/summarize" icon="book">
  Summarize a book from URL, PDF, EPUB, or pasted text
</Card>

Generates a structured summary of book content. Results are cached by content hash to avoid duplicate processing.

### Authentication

Optional. Summaries can be created without authentication.

### Request

Send as `multipart/form-data` with one of the following:

<ParamField body="url" type="string">
  URL to extract content from (article, blog post, etc.)
</ParamField>

<ParamField body="file" type="file">
  PDF or EPUB file to summarize. Maximum 50MB.
  
  Supported content types:
  - `application/pdf`
  - `application/epub+zip`
</ParamField>

<ParamField body="text" type="string">
  Plain text content to summarize. Must be 500-500,000 characters.
</ParamField>

### Response

<ResponseField name="id" type="string">
  Unique identifier for the summary
</ResponseField>

<ResponseField name="cached" type="boolean">
  Whether this summary was retrieved from cache
</ResponseField>

<ResponseField name="metadata" type="object">
  Book metadata extracted or generated
  
  <Expandable title="properties">
    <ResponseField name="title" type="string">
      Book title (extracted or auto-generated)
    </ResponseField>
    
    <ResponseField name="author" type="string | null">
      Author name if detected
    </ResponseField>
    
    <ResponseField name="isAutoGenerated" type="boolean">
      Whether the title was auto-generated
    </ResponseField>
    
    <ResponseField name="language" type="string">
      ISO 639-1 language code (e.g., "en", "am")
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="summary" type="object">
  High-level summary of the book
  
  <Expandable title="properties">
    <ResponseField name="coreThesis" type="string">
      One-sentence central argument (max 30 words)
    </ResponseField>
    
    <ResponseField name="keyInsights" type="array">
      5-7 most important ideas, one sentence each
    </ResponseField>
    
    <ResponseField name="actionableTakeaways" type="array">
      3-5 specific actions readers can take
    </ResponseField>
    
    <ResponseField name="readThisIf" type="string">
      Who should read this and why (1-2 sentences)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="chapters" type="array">
  Array of chapter summaries
  
  <Expandable title="properties">
    <ResponseField name="number" type="integer">
      Chapter number (sequential from 1)
    </ResponseField>
    
    <ResponseField name="title" type="string">
      Chapter title (extracted or generated)
    </ResponseField>
    
    <ResponseField name="summary" type="string">
      Detailed chapter summary with Core Idea, Key Points, Example, and Actionable Takeaway sections
    </ResponseField>
  </Expandable>
</ResponseField>

### Example: Summarize from URL

```bash
curl -X POST https://your-domain.com/api/summarize \
  -F "url=https://example.com/article"
```

```json
{
  "id": "sum_abc123",
  "cached": false,
  "metadata": {
    "title": "The Art of Learning",
    "author": "Josh Waitzkin",
    "isAutoGenerated": false,
    "language": "en"
  },
  "summary": {
    "coreThesis": "Mastery in any field comes from embracing incremental growth through deep focus and resilience.",
    "keyInsights": [
      "Peak performance requires complete presence in the moment",
      "Mistakes are essential feedback for growth",
      "Recovery between intense focus sessions is crucial"
    ],
    "actionableTakeaways": [
      "Practice deliberate focus for 90-minute blocks",
      "Analyze your mistakes without emotional judgment",
      "Build recovery rituals after intense work"
    ],
    "readThisIf": "You want to understand how world-class performers develop mastery and apply it to your own learning journey."
  },
  "chapters": [
    {
      "number": 1,
      "title": "Innocent Moves",
      "summary": "**Core Idea**: Early learning should focus on joy and exploration rather than rigid technique.\n\n**Key Points**:\n- Natural talent emerges when learners feel safe to experiment\n- Pressure from external expectations can stifle growth\n- Foundation built on curiosity outlasts one built on discipline\n\n**Example**: The author's early chess training emphasized creative play over memorizing openings, allowing intuition to develop naturally.\n\n**Actionable Takeaway**: When learning something new, prioritize enjoyment and exploration for the first 30 days before adding structure."
    }
  ]
}
```

### Example: Summarize from file

```bash
curl -X POST https://your-domain.com/api/summarize \
  -F "file=@book.pdf"
```

### Example: Summarize pasted text

```bash
curl -X POST https://your-domain.com/api/summarize \
  -F "text=Your long text content here..."
```

### Validation errors

<ResponseField name="400 Bad Request">
  **No content provided**
  ```json
  {
    "detail": "Provide URL, file (PDF/EPUB) or text content"
  }
  ```
  
  **Unsupported file type**
  ```json
  {
    "detail": "Only PDF and EPUB files are supported"
  }
  ```
  
  **File too large**
  ```json
  {
    "detail": "File too large. Max: 50MB"
  }
  ```
  
  **Text too short**
  ```json
  {
    "detail": "Text too short. Minimum 500 characters."
  }
  ```
</ResponseField>

## Get summary

<Card title="GET /api/summary/{summary_id}" icon="file-lines">
  Retrieve an existing summary by ID
</Card>

Fetch a previously generated summary. If authenticated, includes whether the summary is saved to the user's library.

### Authentication

Optional. Anonymous users can retrieve summaries, but `isSaved` will always be `false`.

### Path parameters

<ParamField path="summary_id" type="string" required>
  The unique identifier of the summary
</ParamField>

### Response

Returns the same structure as POST /summarize, plus:

<ResponseField name="isSaved" type="boolean">
  Whether the current user has saved this summary to their library
</ResponseField>

### Example

```bash
curl https://your-domain.com/api/summary/sum_abc123 \
  -H "Authorization: Bearer <session-token>"
```

```json
{
  "id": "sum_abc123",
  "cached": true,
  "isSaved": true,
  "metadata": { ... },
  "summary": { ... },
  "chapters": [ ... ]
}
```

### Errors

<ResponseField name="404 Not Found">
  ```json
  {
    "detail": "Summary not found"
  }
  ```
</ResponseField>

## Export summary as PDF

<Card title="GET /api/summary/{summary_id}/pdf" icon="file-pdf">
  Download a formatted PDF of the summary
</Card>

Generate and download a PDF export of the summary.

### Authentication

Not required.

### Path parameters

<ParamField path="summary_id" type="string" required>
  The unique identifier of the summary
</ParamField>

### Response

Returns a PDF file with:
- Content-Type: `application/pdf`
- Content-Disposition: `attachment; filename="{title}_summary.pdf"`

### Example

```bash
curl https://your-domain.com/api/summary/sum_abc123/pdf \
  -o summary.pdf
```

### Errors

<ResponseField name="404 Not Found">
  ```json
  {
    "detail": "Summary not found"
  }
  ```
</ResponseField>
