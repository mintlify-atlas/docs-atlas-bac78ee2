title: Uploading books

Reading Assistant supports three different ways to upload and summarize books. Choose the method that works best for your workflow.

## Upload methods

<Steps>
  <Step title="Choose your upload method">
    On the home page, you'll see three tabs:
    
    - **Upload File** - For PDF or EPUB files stored on your device
    - **Paste Link** - For web articles and online book content
    - **Paste Text** - For copying and pasting book content directly
    
    Select the tab that matches your content type.
  </Step>

  <Step title="Provide your content">
    Depending on your chosen method:
    
    **File upload:**
    - Drag and drop a PDF or EPUB file into the upload area
    - Or click to browse and select a file from your device
    - Maximum file size: 50MB
    - Supported formats: PDF, EPUB
    
    **Link:**
    - Paste a URL to an article or book page
    - Must be HTTP or HTTPS
    - Video URLs are not supported
    
    **Text:**
    - Paste the book content directly into the text area
    - Minimum length: varies based on settings
  </Step>

  <Step title="Submit for summarization">
    Click the **Summarize** button to start processing.
    
    The system will:
    1. Extract text content from your file or URL
    2. Check if this content has been summarized before (cached results)
    3. Generate an AI-powered summary with key insights
    4. Redirect you to the summary page automatically
  </Step>
</Steps>

## Validation rules

The system validates your input before processing:

- **Files** must be PDF or EPUB format
- **URLs** must be valid web links (no video platforms)
- **Text** must meet minimum character requirements
- **File size** cannot exceed the configured maximum

<Note>
If you upload the same content twice, Reading Assistant returns the cached summary instantly instead of processing again.
</Note>

## API endpoint

For developers building custom integrations, the upload endpoint accepts multipart form data:

```bash
POST /api/summarize
Content-Type: multipart/form-data

# Form fields (one required):
url: "https://example.com/article"
file: <binary file data>
text: "Pasted content..."
```

**Response:**

```json
{
  "id": "abc123",
  "cached": false,
  "metadata": {
    "title": "Book Title",
    "author": "Author Name",
    "language": "en",
    "isAutoGenerated": true
  },
  "summary": {
    "coreThesis": "Main argument...",
    "keyInsights": ["Insight 1", "Insight 2"],
    "actionableTakeaways": ["Action 1", "Action 2"],
    "readThisIf": "You should read this if..."
  },
  "chapters": [
    {
      "title": "Chapter 1",
      "summary": "Chapter summary..."
    }
  ]
}
```

**Error responses:**

- `400` - Invalid input (no content provided, unsupported format, file too large)
- `500` - Summarization failed

## Content caching

Reading Assistant uses content hashing to detect duplicate submissions:

- Each uploaded book's content is hashed
- If the hash matches an existing summary, it's returned immediately
- This saves processing time and API costs
- The response includes `"cached": true` for cached results

<Tip>
To force a new summary of the same content, the book would need to be modified slightly or you would need to contact support.
</Tip>
