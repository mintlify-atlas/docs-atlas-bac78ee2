# Quick start

This guide will help you get Reading Assistant running locally and generate your first book summary.

## Prerequisites

Before you begin, ensure you have:
- **Python 3.12+** installed
- **Node.js 24+** installed
- **uv** Python package manager ([install guide](https://docs.astral.sh/uv/getting-started/installation/))
- **OpenRouter API key** ([get one here](https://openrouter.ai/))
- **OpenAI API key** ([get one here](https://platform.openai.com/))

## Installation

<Steps>
  <Step title="Clone and setup backend">
    Create a virtual environment and install Python dependencies:

    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    
    uv pip install -r requirements.txt
    ```

    The backend uses these key dependencies:
    - `fastapi==0.128.0` - Web framework
    - `llama-index==0.14.12` - Document processing
    - `chromadb==1.4.1` - Vector storage
    - `pymupdf==1.26.7` - PDF extraction
    - `ebooklib==0.20` - EPUB extraction
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root:

    ```bash
    cp .env.example .env
    ```

    Edit `.env` with your API keys:

    ```bash
    OPENROUTER_API_KEY=your_openrouter_key_here
    OPENAI_API_KEY=your_openai_key_here
    LLM_MODEL=google/gemini-3-flash-preview
    DATABASE_URL=sqlite:///data/app.db
    BETTER_AUTH_URL=http://localhost:3000
    LOG_LEVEL=INFO
    ```

    <Note>
      The SQLite database will be automatically created in the `data/` directory when you first start the server.
    </Note>
  </Step>

  <Step title="Setup frontend">
    Install Node.js dependencies and configure the frontend:

    ```bash
    cd client
    npm install
    ```

    Key frontend dependencies include:
    - `next@16.1.1` - React framework
    - `react@19.2.3` - UI library
    - `better-auth@^1.4.17` - Authentication
    - `drizzle-orm@^0.45.1` - Database ORM
  </Step>

  <Step title="Configure auth database">
    Better Auth requires a PostgreSQL database. Create a Postgres instance (e.g., via [Neon](https://neon.tech/)) and set up the auth database:

    Create `client/.env.local`:

    ```bash
    DATABASE_URL=postgresql://user:password@host/dbname
    NEXT_PUBLIC_API_URL=http://localhost:8000
    ```

    Run database migrations:

    ```bash
    npx drizzle-kit push
    ```
  </Step>

  <Step title="Start the servers">
    Open two terminal windows.

    **Terminal 1 - Backend:**
    ```bash
    uvicorn api:app --reload --host 0.0.0.0 --port 8000
    ```

    **Terminal 2 - Frontend:**
    ```bash
    cd client
    npm run dev
    ```

    <Note>
      The backend API will be available at `http://localhost:8000` and the frontend at `http://localhost:3000`.
    </Note>
  </Step>
</Steps>

## Generate your first summary

Now that everything is running, let's create your first book summary!

<Steps>
  <Step title="Open the application">
    Navigate to [http://localhost:3000](http://localhost:3000) in your browser.
  </Step>

  <Step title="Upload a book">
    You have three options:
    - **Upload a PDF or EPUB file** (max file size configured in settings)
    - **Paste text content** directly (minimum 100 characters)
    - **Provide a URL** to extract content from

    <Warning>
      Files are limited by the `MAX_FILE_SIZE_MB` setting. Text content must be at least `MIN_TEXT_LENGTH` characters.
    </Warning>
  </Step>

  <Step title="Get your summary">
    Click "Summarize" and wait for the AI to process your book. You'll receive:
    - **Core thesis** - The main argument or idea
    - **Key insights** - Important concepts and takeaways
    - **Actionable items** - Practical steps you can apply
    - **Target audience** - Who should read this book
    - **Chapter summaries** - Detailed breakdown of each chapter
  </Step>

  <Step title="Explore additional features">
    Once you have a summary, you can:
    - **Chat with the book** - Ask questions about the content using the chat feature
    - **Export to PDF** - Download a formatted PDF of the summary
    - **Save to library** - Add the summary to your personal collection

    <CodeGroup>
    ```bash API example
    # Get a summary by ID
    curl http://localhost:8000/api/summary/{summary_id}

    # Download as PDF
    curl http://localhost:8000/api/summary/{summary_id}/pdf -o summary.pdf

    # Chat with the book
    curl -X POST http://localhost:8000/api/chat \
      -H "Content-Type: application/json" \
      -d '{"summary_id": "your_id", "message": "What are the main themes?"}'
    ```

    ```python Python example
    import httpx

    # Summarize a book
    async with httpx.AsyncClient() as client:
        with open("book.pdf", "rb") as f:
            response = await client.post(
                "http://localhost:8000/api/summarize",
                files={"file": f}
            )
            summary = response.json()
            print(f"Summary ID: {summary['id']}")
    ```
    </CodeGroup>
  </Step>
</Steps>
