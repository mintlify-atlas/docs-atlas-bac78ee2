Reading Assistant uses OpenRouter to access various language models for generating book summaries. This page explains how to configure and customize LLM behavior.

## Model configuration

The LLM model is configured in `core/config.py:18`:

```python
LLM_MODEL: str = os.getenv("LLM_MODEL", "google/gemini-3-flash-preview")
```

### Choosing a model

You can use any model available on [OpenRouter](https://openrouter.ai/models). Set the `LLM_MODEL` environment variable to the model identifier:

```bash
LLM_MODEL=google/gemini-3-flash-preview
```

**Recommended models:**

- **google/gemini-3-flash-preview** (default) - Fast and cost-effective for most books
- **anthropic/claude-3.5-sonnet** - Higher quality summaries, more expensive
- **openai/gpt-4** - Excellent quality, higher cost
- **meta-llama/llama-3.1-70b-instruct** - Good balance of quality and cost

<Warning>
  Some models have token limits that may be too small for very long books. The default model (Gemini 3 Flash) handles up to 400,000 characters of input.
</Warning>

## API configuration

The OpenRouter API endpoint is configured in `core/config.py:14`:

```python
OPENROUTER_BASE_URL: str = "https://openrouter.ai/api/v1/chat/completions"
```

This typically doesn't need to be changed unless you're using a proxy or custom endpoint.

## Generation parameters

The summarization request includes these parameters in `core/summarizer.py:97-105`:

```python
payload = {
    "model": settings.LLM_MODEL,
    "messages": [
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user", "content": text},
    ],
    "temperature": 0.3,  # Lower = more consistent
    "max_tokens": 5000,
}
```

### Temperature

The `temperature` is set to `0.3` for consistency. Lower values make the model more deterministic and focused.

- **0.0-0.3**: Very consistent, good for structured output
- **0.4-0.7**: Balanced creativity and consistency
- **0.8-1.0**: More creative, less predictable

### Max tokens

The `max_tokens` is set to `5000`, which limits the length of the generated summary. This is sufficient for most books with multiple chapters.

### Timeout

The HTTP request timeout is set to 120 seconds in `core/summarizer.py:115`:

```python
async with httpx.AsyncClient(timeout=120.0) as client:
```

## System prompt

The system prompt defines the structure and rules for summary generation. It's defined in `core/summarizer.py:19-82`:

<Accordion title="View full system prompt">
```
You are a reading assistant. Analyze the provided text and return a JSON object with this exact structure:

IMPORTANT: Detect the language of the provided text and return all summaries in that same language.
- If the book is in Amharic, write summaries in Amharic
- If the book is in English, write summaries in English
- And so on for any language

Only the JSON keys should remain in English. 
All content values (title, author, summary, chapter summaries) should be in the detected language.

{
  "metadata": {
    "title": "string",
    "author": "string or null",
    "isAutoGenerated": boolean
    "language": "string"
  },
  "summary": {
    "coreThesis": "string",
    "keyInsights": ["string array, 5-7 items"],
    "actionableTakeaways": ["string array, 3-5 items"],
    "readThisIf": "string"
  },
  "chapters": [
    { "number": 1, "title": "string", "summary": "string" }
  ]
}

RULES:

Metadata:
- Extract title/author if visible in text (title pages, headers, "by [name]")
- If not found: generate descriptive title, set author to null
- isAutoGenerated = true only if you created the title
- language: ISO 639-1 language code of the detected language (e.g., "en", "am")

Summary:
- coreThesis: One sentence (max 30 words), the central argument
- keyInsights: 5-7 most important ideas, one sentence each
- actionableTakeaways: 3-5 specific actions reader can take, start with verbs
- readThisIf: 1-2 sentences on who should read this and why

Chapters:
- Extract all chapters/sections from the text (in original text order)
- If no clear structure exists, create 3-7 thematic sections with generated titles
- Each chapter summary should include (with blank lines between each section):

  **Core Idea**: 1-2 sentences on the main argument

  **Key Points**: 2-3 bullet points on reasoning/evidence

  **Example**: One concrete example or case study from the chapter

  **Actionable Takeaway**: One specific thing the reader can apply immediately

- Use this exact format with blank lines separating each section
- Return flat list, no grouping by parts
- Skip prefaces, acknowledgments, indexes
- If chapters are extracted from the text, preserve original numbering
- If chapters are generated, number sequentially starting from 1 in reading order

Return ONLY valid JSON. No markdown, no explanation.
```
</Accordion>

### Customizing the prompt

To modify the summary structure or rules:

1. Edit the `SYSTEM_PROMPT` variable in `core/summarizer.py`
2. Restart the backend server to apply changes

<Warning>
  Changing the prompt structure may break the JSON parsing logic in `core/summarizer.py:144-174`. Ensure your changes maintain valid JSON output.
</Warning>

## Text truncation

If the input text exceeds 400,000 characters, it's automatically truncated in `core/summarizer.py:92-95`:

```python
max_content_chars = 400_000
if len(text) > max_content_chars:
    logger.warning(f"Text truncated from {len(text)} to {max_content_chars} chars")
    text = text[:max_content_chars]
```

This prevents exceeding model context limits and API timeouts.

## Error handling

The summarizer handles these error cases:

- **Missing API key**: Raises `SummaryError` before making the request
- **API timeout**: 120-second timeout with clear error message
- **Invalid JSON**: Attempts to extract JSON from markdown code blocks
- **HTTP errors**: Logs full error response from OpenRouter

All errors are logged to `logs/app.log` for debugging.

## Embeddings for RAG

The chat feature uses OpenAI embeddings for vector search. This is configured in `core/config.py:39`:

```python
EMBEDDING_MODEL = "text-embedding-3-small"
```

### Embedding parameters

```python
CHUNK_SIZE = 1024        # Characters per chunk
CHUNK_OVERLAP = 200      # Overlap between chunks
TOP_K = 5                # Number of chunks to retrieve
```

These settings control how book content is split and retrieved for chat responses. See `core/config.py:40-42` for details.
